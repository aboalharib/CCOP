# 1 Purpose

This document defines the geospatial architecture used by the Climate Change Operation Platform (CCOP).

The purpose of this architecture is to provide map-based situational awareness for environmental monitoring operations.

---

# 2 Scope

The geospatial architecture supports:

- monitoring station visualization
- environmental data mapping
- geospatial alert visualization
- operational situational awareness
- environmental trend visualization

The geospatial layer consumes processed telemetry generated by the **CCOP Climate Intelligence Platform (CIP)**.

---

# 3 Architectural Position

The geospatial architecture represents the **Geospatial Operations Layer** within the CCOP Hybrid Platform Architecture.


Monitoring & Telemetry Layer
↓
Climate Intelligence Platform (CIP)
↓
Geospatial Operations Layer
↓
Operational Dashboards


This layer converts processed environmental data into visual operational intelligence.

---

# 4 Geospatial Data Sources

The geospatial layer receives data from:

| Source | Description |
|------|-------------|
CIP Processed Telemetry | validated environmental measurements |
Station Metadata | location and device information |
Alert Service | environmental alerts |
Analytics Service | environmental indicators |

---

# 5 Monitoring Station Visualization

The geospatial interface visualizes the CCOP monitoring network.

| Station Type | Map Representation |
|---|---|
TerraNode | reference station symbol |
TerraEdge | distributed station symbol |
TerraAir | portable station symbol |

Stations are displayed on the map using their geospatial coordinates.

---

# 6 Geospatial Data Model

Each monitoring station includes the following geospatial attributes:

| Field | Description |
|---|---|
station_id | unique identifier |
latitude | geographic latitude |
longitude | geographic longitude |
monitoring_role | reference / distributed / portable |
status | operational status |

These attributes allow geospatial queries and visualization.

---

# 7 Map Layers

The geospatial architecture supports multiple map layers.

| Layer | Description |
|---|---|
Station Layer | monitoring station locations |
Environmental Data Layer | environmental measurements |
Alert Layer | environmental alerts |
Trend Layer | historical environmental trends |

These layers can be enabled or disabled by the operator.

# 8 Environmental Data Visualization

Environmental telemetry can be visualized as:

- point measurements
- trend graphs
- heat maps
- spatial overlays

Visualization methods may include:

- color-coded station indicators
- time-series charts
- geospatial overlays

---

# 9 Alert Visualization

The geospatial interface displays environmental alerts generated by CIP.

Alert types may include:

- pollution threshold exceedance
- telemetry anomalies
- station communication failures

Alerts are displayed as map markers or highlighted regions.

---

# 10 Geospatial Query Capability

Operators may perform spatial queries such as:

- nearby station search
- region-based monitoring analysis
- alert proximity analysis

These queries rely on the geospatial database capabilities of the backend.

---

# 11 Geospatial Database Integration

The geospatial layer uses spatial database support.

Recommended technology:

| Component | Technology |
|---|---|
Geospatial Database | PostGIS |
Mapping Library | Leaflet |
Base Map | OpenStreetMap |

PostGIS enables spatial queries and environmental mapping.

---

# 12 Geospatial APIs

The CIP backend exposes APIs that support the geospatial layer.

Example APIs include:

| Endpoint | Purpose |
|---|---|
/stations | list monitoring stations |
/stations/{id} | station details |
/stations/{id}/latest | latest telemetry |
/alerts | active alerts |
/layers/environment | environmental overlays |

These APIs allow the dashboard to display real-time environmental conditions.
