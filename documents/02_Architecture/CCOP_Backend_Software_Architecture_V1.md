# 1 Purpose

This document defines the backend software architecture of the Climate Change Operation Platform (CCOP).

The purpose of this document is to describe the backend service structure required to ingest, validate, calibrate, govern, store, analyze, and distribute environmental telemetry generated by the CCOP monitoring network.

---

# 2 Scope

This document defines the backend architecture supporting:

- TerraNode telemetry ingestion
- TerraEdge telemetry ingestion
- TerraAir telemetry ingestion
- telemetry validation
- calibration processing
- environmental data governance
- environmental data storage
- alert generation
- reporting generation
- dashboard data services
- simulation telemetry ingestion

# 4 Architectural Principles

The CCOP backend software architecture follows these principles:

- service-oriented backend structure
- clear separation of responsibilities
- scalable telemetry ingestion
- traceable data processing
- calibration-aware telemetry handling
- governance-aware dataset management
- support for simulation and live telemetry through the same backend pathways

The CCOP backend is organized into logical services.

```text
Monitoring Stations / Simulators
          ↓
Telemetry Ingestion API
          ↓
Raw Stream / Queue
          ↓
Validation Service
          ↓
Calibration Service
          ↓
Governance Service
          ↓
Processed Data Store
          ↓
Analytics / Alerts / Reporting
          ↓
Dashboard APIs / External Outputs
6 Backend Service Domains

The backend architecture is divided into the following service domains:

Service Domain	Purpose
Ingestion Services	receive telemetry
Validation Services	verify telemetry correctness
Calibration Services	apply calibration logic
Governance Services	assign validation and traceability metadata
Storage Services	persist raw and processed data
Analytics Services	derive trends and indicators
Alert Services	detect anomalies and threshold events
Reporting Services	produce structured reporting outputs
Dashboard API Services	provide frontend data access
7 Telemetry Ingestion Service

The telemetry ingestion service is responsible for receiving telemetry from monitoring stations and simulators.

Responsibilities

receive JSON telemetry payloads

authenticate and identify station source

verify minimum required message structure

stamp message receipt metadata

publish messages to raw telemetry stream

Inputs

TerraNode telemetry

TerraEdge telemetry

TerraAir telemetry

simulation telemetry

Outputs

raw telemetry stream

ingestion log

8 Validation Service

The validation service verifies that telemetry messages are structurally and logically acceptable.

Responsibilities

required field validation

data type validation

value range validation

timestamp validation

geographic coordinate validation

station role validation

Possible results

accepted

accepted with warning

rejected

Outputs

validated telemetry

validation status

validation reason codes

9 Calibration Service

The calibration service applies measurement adjustment logic where required.

Responsibilities

compare distributed station values against TerraNode reference values where applicable

apply calibration adjustments

attach calibration metadata

preserve original raw values where required for traceability

Calibration inputs

validated telemetry

TerraNode reference data

calibration models

calibration configuration rules

Outputs

calibrated telemetry

calibration metadata

10 Governance Service

The governance service ensures telemetry is traceable and suitable for downstream use.

Responsibilities

assign data quality status

assign dataset version metadata

record validation timestamp

apply governance policy identifiers

mark dataset suitability for reporting and analytics

Outputs

governance metadata

reportable / non-reportable status

AI-ready / non-AI-ready status

11 Storage Architecture

The backend shall maintain separate logical storage layers.

11.1 Raw Telemetry Store

Stores telemetry exactly as received for traceability and audit.

11.2 Processed Telemetry Store

Stores validated, calibrated, and governed telemetry.

11.3 Metadata Store

Stores:

monitoring station metadata

calibration metadata

governance metadata

configuration data

11.4 Reporting Dataset Store

Stores structured environmental datasets prepared for reporting.

Responsibilities

environmental indicator generation

analytics datasets

derived metrics

16 Simulation Compatibility

The backend architecture shall support simulation telemetry using the same ingestion and processing paths as live telemetry.

Simulation compatibility requirements include:

same telemetry schema

same validation logic

same calibration metadata model

same governance metadata model

same dashboard outputs

This ensures that the simulation environment is a true pre-pilot validation environment.
