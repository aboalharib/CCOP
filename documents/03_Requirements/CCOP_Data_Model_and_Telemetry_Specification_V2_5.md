# 1 Purpose

This document defines the official data model and telemetry structure used by the Climate Change Operation Platform (CCOP).

The purpose of this specification is to establish a consistent schema for telemetry messages generated by monitoring devices and processed by the CCOP Climate Intelligence Platform (CIP).

---

# 2 Scope

This specification applies to telemetry generated by:

- TerraNode monitoring stations
- TerraEdge monitoring stations
- TerraAir portable monitoring devices

The schema supports:

- environmental telemetry
- meteorological data
- system health monitoring
- sensor lifecycle monitoring
- calibration metadata

# 4 Telemetry Architecture

Telemetry generated by monitoring devices follows the CCOP data pipeline.


Monitoring Device
↓
Secure Transmission (WireGuard VPN)
↓
Message Broker (Kafka / Pravega)
↓
Stream Processing (Apache Flink)
↓
Time-Series Storage
↓
CCOP Climate Intelligence Platform (CIP)


---

# 5 Core Telemetry Fields

Each telemetry message must include the following core fields.

| Field | Description |
|---|---|
device_id | unique device identifier |
timestamp | UTC timestamp |
device_type | TerraNode / TerraEdge / TerraAir |
latitude | geographic latitude |
longitude | geographic longitude |

# 8 System Health Telemetry

Device health information supports infrastructure monitoring.

| Field | Description |
|---|---|
cpu_usage | CPU load |
memory_usage | RAM usage |
power_input | input voltage |
battery_voltage | battery level |
internal_temperature | enclosure temperature |
network_status | connectivity status |

These metrics are also transmitted to Zabbix.

---

# 9 Sensor Lifecycle Data Model

TerraNode and TerraEdge devices include lifecycle telemetry for electrochemical sensors.

Lifecycle fields include:

| Field | Description |
|---|---|
sensor_name | sensor model identifier |
parameter | measured gas |
sensor_serial | sensor serial number |
install_date | sensor installation date |
lifespan_days | expected operational lifespan |
days_used | days since installation |
days_remaining | estimated days remaining |
life_percent | percentage of life remaining |
lifecycle_status | operational lifecycle state |

---

# 10 Lifecycle Status Values

Lifecycle status codes are defined as follows.

| Code | Status |
|---|---|
0 | OK |
1 | WARNING |
2 | CRITICAL |
3 | EXPIRED |

These values support automated maintenance alerts.

12 Data Storage Model

Telemetry data is stored in two datasets.

Dataset	Description
Raw Data	original measurements from devices
Processed Data	validated and corrected measurements

Raw data must never be overwritten.

13 Data Governance Integration

Telemetry data must integrate with the CCOP Environmental Data Governance Framework.

This ensures:

data integrity

traceability

calibration transparency

reproducibility of environmental measurements

14 Version Control

All telemetry schema updates must be version controlled.

Backward compatibility should be maintained whenever possible.
