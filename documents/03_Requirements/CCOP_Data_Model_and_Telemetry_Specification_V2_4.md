# 1 Purpose

This document defines the **data model and telemetry specification** for the Climate Change Operation Platform (CCOP).

The specification describes how environmental telemetry is structured, encoded, transmitted, stored, validated, and processed within the system.

---

# 2 Scope

This specification applies to telemetry generated by:

- TerraNode monitoring stations  
- TerraEdge monitoring stations  
- TerraAir monitoring stations  

The specification defines the structure of environmental data transmitted to the CCOP Climate Intelligence Platform.

# 4 Data Model Overview

The CCOP data model represents environmental observations collected by distributed monitoring stations.

Each telemetry message represents a **timestamped environmental observation event**.

Telemetry records include:

- station identification  
- monitoring role  
- timestamp  
- geospatial location  
- environmental measurements  
- calibration metadata  
- data governance metadata  
- system health information  

---

# 5 Monitoring Station Identification

Each monitoring station has a unique identifier.

Format:


CCOP-[TYPE]-[NUMBER]


Examples:


CCOP-TN-001
CCOP-TE-015
CCOP-TA-007


| Code | Station |
|------|--------|
| TN | TerraNode |
| TE | TerraEdge |
| TA | TerraAir |

---

# 6 Monitoring Role Field

Telemetry messages include a monitoring role field reflecting the Environmental Monitoring Pyramid.

| Value | Meaning |
|------|--------|
| reference | TerraNode |
| distributed | TerraEdge |
| portable | TerraAir |

---

# 7 Timestamp Model

Telemetry timestamps use **ISO 8601 format**.

Example:


2026-05-01T14:22:30Z


The timestamp represents the moment the environmental observation was recorded.

---

# 8 Geospatial Metadata

Monitoring stations provide location information.

| Field | Description |
|------|-------------|
| latitude | decimal degrees |
| longitude | decimal degrees |

Example:


latitude: 25.2854
longitude: 51.5310


Geospatial metadata supports mapping and spatial environmental analysis.

# 10 Calibration Metadata

Calibration metadata supports environmental data quality management.

| Field | Description |
|------|-------------|
| calibration_status | sensor calibration state |
| calibration_date | last calibration timestamp |
| calibration_reference | reference station used |

Example:


calibration_reference: CCOP-TN-001


---

# 11 Environmental Data Governance Metadata

Telemetry may include governance metadata supporting dataset traceability.

| Field | Description |
|------|-------------|
| data_quality_status | dataset validation state |
| dataset_version | dataset version identifier |
| validation_timestamp | last validation time |
| governance_policy | governance rule applied |

These fields support environmental dataset credibility.

---

# 12 Telemetry Message Structure

Telemetry messages include the following fields.

| Field | Description |
|------|-------------|
| station_id | monitoring station identifier |
| device_type | TerraNode / TerraEdge / TerraAir |
| monitoring_role | reference / distributed / portable |
| timestamp | observation time |
| location | geographic coordinates |
| measurements | environmental measurements |
| calibration | calibration metadata |
| governance | governance metadata |
| system_status | device health information |

---
